# Técnicas de Relevamiento

Guía práctica para recopilar información sobre el sistema actual y las necesidades del nuevo sistema.

## ¿Qué es el Relevamiento?

Es el proceso de **recopilación, descubrimiento y estudio** de información sobre el negocio. El analista actúa como investigador para entender **QUÉ** debe hacer el sistema, sin preocuparse todavía por **CÓMO** hacerlo.

## Principales Técnicas de Relevamiento

### Entrevistas

**Objetivo**: Obtener información directa de personas clave (stakeholders, usuarios).

**Tipos**:
- **Estructurada**: preguntas predefinidas, mismo orden para todos
- **Semiestructurada**: guía de temas, pero flexibilidad en el orden
- **No estructurada**: conversación abierta, exploración libre

**Cuándo usar**: Para entender necesidades, problemas y visiones del negocio.

**Consejos**:
- Preparar preguntas abiertas ("¿Cómo es el proceso de...?")
- Evitar preguntas que sugieran la respuesta
- Escuchar más que hablar
- Documentar durante la conversación
- Hacer seguimiento a respuestas ambiguas

---

### Observación Directa

**Objetivo**: Ver cómo realmente funciona el negocio, no cómo debería funcionar.

**Proceso**:
1. Acompañar a usuarios en sus tareas cotidianas
2. Registrar actividades, procedimientos y excepciones
3. Identificar cuellos de botella y problemas reales

**Cuándo usar**: Cuando hay discrepancia entre lo que dicen y lo que hacen.

**Resultado**: Detectar conflictos o necesidades desde la "empatía" de ponerse en los zapatos del otro usuario.

---

### Revisión de Documentación Existente

**Objetivo**: Recopilar información de documentos, políticas, manuales.

**Fuentes**:
- Manuales de procedimientos
- Políticas internas
- Registros históricos
- Reportes actuales
- Sistemas legados

**Cuándo usar**: Para entender el contexto histórico y políticas establecidas.

---

### Cuestionarios y Encuestas

**Objetivo**: Recopilar información de múltiples personas de forma escalable.

**Ventajas**:
- Llega a muchas personas
- Respuestas comparables
- Económico

**Desventajas**:
- Menos profundidad que entrevistas
- Baja tasa de respuesta

**Cuándo usar**: Para validar hipótesis o recopilar datos de grupos grandes.

---

### Talleres o Sesiones de Grupo (Workshops)

**Objetivo**: Reunir a múltiples stakeholders para discutir y consensuar.

**Estructura**:
1. Presentación del tema
2. Discusión abierta
3. Lluvia de ideas
4. Consenso sobre puntos clave

**Cuándo usar**: Para alinear visiones, resolver conflictos, generar ideas en grupo.

---

### Análisis de Procesos Actuales

**Objetivo**: Mapear y entender los procesos de negocio existentes.

**Herramientas**:
- Diagramas de flujo
- Diagramas de actividad
- Timeline de procesos

**Cuándo usar**: Para identificar problemas y oportunidades de mejora.

---

### Prototipado o Mockups

**Objetivo**: Mostrar visualmente cómo podría funcionar el nuevo sistema.

**Tipos**:
- Prototipos en papel
- Mockups digitales
- Prototipos interactivos

**Cuándo usar**: Para validar entendimiento con usuarios antes de desarrollar.

---

### Análisis Comparativo (Benchmarking)

**Objetivo**: Estudiar cómo otros sistemas o empresas resuelven problemas similares.

**Aplicación**:
- Productos competidores
- Mejores prácticas de la industria
- Sistemas en otras empresas

**Cuándo usar**: Para inspirar soluciones e identificar estándares.

---

## Plantilla de Minuta de Relevamiento

```markdown
## Minuta de Relevamiento
Fecha: DD/MM/YYYY
Asistentes: [Lista de personas]
Tema: [Descripción del relevamiento]

### Información recopilada
- Punto 1: [Descripción]
- Punto 2: [Descripción]
- Punto 3: [Descripción]

### Problemas identificados
- Problema 1: [Descripción y impacto]
- Problema 2: [Descripción e impacto]

### Preguntas abiertas
- Pregunta 1: [A resolver]
- Pregunta 2: [A resolver]

### Próximos pasos
- Acción 1: [Responsable y fecha]
- Acción 2: [Responsable y fecha]

### Anexos
- Documentación recopilada
- Diagramas/fotos
- Referencias
```

## Checklist de Buenas Prácticas

- [ ] Se definió objetivo claro para cada sesión de relevamiento
- [ ] Se identificaron y consultaron los stakeholders clave
- [ ] Se documentó toda la información recopilada
- [ ] Se validó la información con los interesados
- [ ] Se identificaron diferencias entre "lo ideal" y "lo real"
- [ ] Se detectaron problemas y oportunidades
- [ ] Se documentaron preguntas abiertas a resolver
- [ ] Se generó una minuta con conclusiones
- [ ] Se estableció un plan de seguimiento
